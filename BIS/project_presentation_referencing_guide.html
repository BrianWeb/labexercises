Referencing Guide for Technology Evaluation Presentation

Students must demonstrate evidence of real-world research using credible sources such as vendor websites, product documentation, analyst reports, and industry publications.

Full APA formatting is not required; however, sources must be credible, relevant, and clearly identifiable (see below).

Marks may be reduced where referencing is missing, unclear, or overly reliant on weak sources.

Web Links Example Slide

Slide Title: Cloud Infrastructure Cost Features

  • Auto scaling adjusts compute capacity dynamically [1].
  • Reserved instances can significantly reduce long-term costs [2].
  • Pay-as-you-go pricing improves flexibility for SMEs [1].

Slide Footer

[1] Amazon Web Services. EC2 Pricing and Auto Scaling.
https://aws.amazon.com/ec2/pricing/

[2] Microsoft Azure. Reserved VM Instances.
https://azure.microsoft.com/pricing/reserved-vm-instances/

At the bottom of slide, as shown in the example above, include the company name, page title (if known), and working web link.

Articles / Books Example

Slide Title: Cybersecurity Risk Trends

  • Human error remains one of the leading causes of security breaches [1].
  • Organisations are increasingly adopting Zero Trust models to reduce risk [2].
  • Insider threats continue to be difficult to detect and manage [1].

Slide Footer

[1] Verizon. (2024). Data Breach Investigations Report.
https://www.verizon.com/business/resources/reports/dbir/

[2] Wolff, J. (2022). Cybersecurity Policy and Practice. University of California Press.

If you’ve already added the references to the last slide in your presentation (instead of adding them on each slide), that’s fine, no need to go back and add it to each slide.

Reuse the same number for the same source. As above, if you use AWS again later, it should still be [1], not a new number.

When to use a Reference

Is it a generally known fact? If yes, then there is no need to reference anything.
Example: “More RAM can increase the amount of computer computations”.

If on the other hand, you include specifications, prices, features, statistics, or vendor claims, or if someone could ask, “How do you know that?” – then add a reference.

Use credible real-world sources

  • Vendor websites, product documentation, analyst reports, and reputable tech news are best.
  • Wikipedia can help you understand topics but should not usually be your main source.
